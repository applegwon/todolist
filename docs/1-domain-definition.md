## 1. 프로젝트 개요

TodoList는 인증된 사용자가 개인별 할일을 등록, 조회, 수정, 삭제할 수 있는 웹 기반 할일 관리 애플리케이션이다. 카테고리 분류와 시작일/종료일 기반의 일정 관리를 핵심 기능으로 제공한다.

---

## 2. 문제 정의

| 문제 | 설명 |
|------|------|
| 개인 인증 기반 할일 관리 부재 | 사용자별로 독립된 할일 데이터를 관리할 수 있는 수단이 없다 |
| 일정 기간 관리 기능 부재 | 할일에 시작일과 종료일을 지정하여 기간 단위로 관리할 수 없다 |

## 3. 핵심 도메인 개념 (엔티티 및 속성)

### 3.1 사용자 (User)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | 고유 식별자 | 시스템 내 사용자 고유 ID |
| 이메일 | 문자열 | 로그인 식별자, 중복 불가 |
| 비밀번호 | 문자열 | 암호화하여 저장 |
| 이름 | 문자열 | 사용자 표시 이름 |
| 테마 | 열거형 | UI 테마 설정 (light / dark), 기본값: light |
| 언어 | 열거형 | UI 언어 설정 (ko / en), 기본값: ko |
| 가입일시 | 일시 | 계정 생성 시각 |

### 3.2 카테고리 (Category)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | 고유 식별자 | 카테고리 고유 ID |
| 이름 | 문자열 | 카테고리 명칭 |
| 사용자 | User | 카테고리 소유 사용자 |

> 시스템은 모든 사용자에게 공통으로 적용되는 **'기본'** 카테고리를 제공한다.

### 3.3 할일 (Todo)

| 속성 | 타입 | 필수 여부 | 설명 |
|------|------|-----------|------|
| id | 고유 식별자 | 필수 | 할일 고유 ID |
| 제목 | 문자열 | 필수 | 할일 내용 요약 |
| 설명 | 문자열 | 선택 | 할일 상세 내용 |
| 시작일자 | 날짜 | 선택 | 할일 시작 날짜 |
| 종료일자 | 날짜 | 선택 | 할일 완료 목표 날짜 |
| 상태 | 열거형 | 필수 | 미시작 / 진행중 / 완료 (저장 속성) |
| 기한초과 여부 | 불리언 | Derived | 종료일자 < 오늘 AND 상태 ≠ 완료 일 때 true (계산된 속성, 저장하지 않음) |
| 카테고리 | Category | 필수 | 분류 카테고리 (미지정 시 '기본' 자동 적용) |
| 사용자 | User | 필수 | 할일 소유 사용자 |
| 등록일시 | 일시 | 필수 | 할일 생성 시각 |
| 수정일시 | 일시 | 필수 | 마지막 수정 시각 |

> **기한초과 여부**는 Derived Attribute(계산된 속성)이다. 별도 컬럼으로 저장하지 않으며, 조회 시점의 날짜와 종료일자를 비교하여 동적으로 계산한다.

## 4. 도메인 규칙 (비즈니스 규칙)

> BR 번호 체계: 인증(BR-1xx), 사용자(BR-2xx), 카테고리(BR-3xx), 할일(BR-4xx)

### 4.1 인증 규칙

- **BR-101** 회원가입 후 로그인(인증)을 완료한 사용자만 앱을 사용할 수 있다.
- **BR-102** 이메일은 전체 시스템에서 유일해야 한다.

### 4.2 사용자 규칙

- **BR-201** 로그인한 사용자는 자신의 프로필 정보(이름, 비밀번호 등)를 수정할 수 있다.
- **BR-202** 사용자는 타인의 정보를 조회하거나 수정할 수 없다.
- **BR-203** 로그인한 사용자는 테마(light/dark)를 선택할 수 있으며, 선택값은 사용자 정보에 저장된다.
- **BR-204** 로그인한 사용자는 언어(ko/en)를 선택할 수 있으며, 선택값은 사용자 정보에 저장된다.
- **BR-205** 로그인 시 저장된 테마와 언어 설정이 자동 적용된다.

### 4.3 카테고리 규칙

- **BR-301** 할일 등록 시 카테고리를 지정하지 않으면 자동으로 '기본' 카테고리가 적용된다.
- **BR-302** '기본' 카테고리는 삭제하거나 수정할 수 없다.
- **BR-303** 사용자는 동일한 이름의 카테고리를 중복 생성할 수 없다.
- **BR-304** 카테고리를 삭제하면 해당 카테고리에 속한 모든 할일의 카테고리는 '기본'으로 변경된다.
- **BR-305** 카테고리의 소유 사용자만 해당 카테고리를 수정하거나 삭제할 수 있다.

### 4.4 할일 규칙

- **BR-401** 할일은 반드시 인증된 사용자에게 귀속된다.
- **BR-402** 종료일자는 시작일자보다 이전일 수 없다. (종료일자 ≥ 시작일자)
- **BR-403** 할일의 등록, 수정, 삭제는 해당 할일의 소유 사용자만 수행할 수 있다.

### 4.5 할일 상태 규칙

**상태(state)** 는 사용자가 명시적으로 설정하는 저장 속성이며, **기한초과 여부**는 조회 시 동적으로 계산되는 Derived Attribute이다.

| 상태 | 조건 |
|------|------|
| 미시작 | 시작일자 > 오늘(today) 이거나, 시작일자가 미지정이고 완료되지 않은 경우 |
| 진행중 | 시작일자 ≤ 오늘(today) 이고, 종료일자 ≥ 오늘(today) 이며, 완료되지 않은 경우 |
| 완료 | 사용자가 명시적으로 완료 처리한 경우 |

> **날짜 경계값 기준:**
> - "시작일자 ≤ 오늘" : 시작일자가 오늘인 경우 **포함(≥ 기준)**하며, 해당 할일은 당일부터 **진행중** 으로 간주한다.
> - "종료일자 ≥ 오늘" : 종료일자가 오늘인 경우 **포함(≥ 기준)**하며, 당일은 아직 기한이 남은 것으로 간주한다.
> - 기한초과 판정: 종료일자 < 오늘(today), 즉 종료일자가 어제 이전인 경우에만 기한초과로 계산한다.

**[기한초과 여부 - Derived Attribute]**

| 계산 조건 | 결과 |
|-----------|------|
| 종료일자 < 오늘(today) AND 상태 ≠ 완료 | true (기한초과) |
| 그 외 | false |

## 5. 주요 유스케이스

### UC-01. 회원가입
- **액터:** 비인증 사용자
- **목적:** 앱 사용을 위한 계정 생성
- **흐름:** 이메일, 비밀번호, 이름 입력 → 이메일 중복 확인 → 계정 생성
- **관련 규칙:** BR-101, BR-102

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 시스템에 존재하지 않는 이메일과 유효한 비밀번호, 이름을 준비한다 | 회원가입 요청을 한다 | 계정이 생성되고 성공 응답을 반환한다 |
| 2 | 이미 가입된 이메일을 준비한다 | 동일 이메일로 회원가입 요청을 한다 | 오류 응답(이메일 중복)을 반환하고 계정이 생성되지 않는다 |
| 3 | 이메일, 비밀번호, 이름 중 하나라도 누락된 값을 준비한다 | 회원가입 요청을 한다 | 오류 응답(필수 항목 누락)을 반환한다 |

---

### UC-02. 로그인
- **액터:** 비인증 사용자
- **목적:** 본인 계정으로 인증하여 앱 접근
- **흐름:** 이메일, 비밀번호 입력 → 인증 성공 → 앱 진입
- **관련 규칙:** BR-101

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 가입된 이메일과 올바른 비밀번호를 준비한다 | 로그인 요청을 한다 | 인증 토큰을 반환하고 앱에 진입한다 |
| 2 | 가입된 이메일과 틀린 비밀번호를 준비한다 | 로그인 요청을 한다 | 오류 응답(인증 실패)을 반환한다 |
| 3 | 존재하지 않는 이메일을 준비한다 | 로그인 요청을 한다 | 오류 응답(인증 실패)을 반환한다 |

### UC-03. 내 정보 수정
- **액터:** 인증된 사용자
- **목적:** 본인 프로필 정보 변경 (이름, 비밀번호, 테마, 언어)
- **흐름:** 수정할 정보 입력 → 저장
- **관련 규칙:** BR-201, BR-202, BR-203, BR-204, BR-205

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 인증된 사용자가 변경할 이름을 준비한다 | 내 정보 수정 요청을 한다 | 이름이 변경되고 성공 응답을 반환한다 |
| 2 | 인증되지 않은 상태이다 | 내 정보 수정 요청을 한다 | 오류 응답(인증 필요)을 반환한다 |
| 3 | 인증된 사용자가 테마를 dark로 변경한다 | 내 정보 수정 요청을 한다 | 테마가 dark로 저장되고 즉시 UI에 반영된다 |
| 4 | 인증된 사용자가 언어를 en으로 변경한다 | 내 정보 수정 요청을 한다 | 언어가 en으로 저장되고 즉시 UI 전체에 반영된다 |
| 5 | 인증된 사용자가 로그인한다 | 로그인 후 앱에 진입한다 | 이전에 저장된 테마와 언어 설정이 자동 적용된다 |

### UC-04. 할일 등록
- **액터:** 인증된 사용자
- **목적:** 새로운 할일 생성
- **흐름:** 제목, 설명(선택), 카테고리(선택), 시작일자(선택), 종료일자(선택) 입력 → 저장
- **관련 규칙:** BR-301, BR-401, BR-402, BR-403

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 인증된 사용자가 제목만 입력하고 카테고리를 지정하지 않는다 | 할일 등록 요청을 한다 | 할일이 생성되고 카테고리는 '기본'으로 저장된다 |
| 2 | 인증된 사용자가 시작일자를 2026-06-01, 종료일자를 2026-05-31로 입력한다 | 할일 등록 요청을 한다 | 오류 응답(종료일자가 시작일자보다 이전)을 반환한다 |
| 3 | 인증된 사용자가 제목을 입력하지 않는다 | 할일 등록 요청을 한다 | 오류 응답(필수 항목 누락)을 반환한다 |
| 4 | 인증된 사용자가 유효한 카테고리 ID와 제목을 입력한다 | 할일 등록 요청을 한다 | 할일이 지정된 카테고리로 생성된다 |

### UC-05. 할일 목록 조회 및 필터링
- **액터:** 인증된 사용자
- **목적:** 본인의 할일 목록 확인
- **필터 조건:**
  - 카테고리별
  - 상태별: 미시작 / 진행중 / 완료
  - 기한초과 (종료일자 < 오늘 AND 상태 ≠ 완료)
- **관련 규칙:** BR-202, BR-401, BR-403

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 인증된 사용자가 본인의 할일이 5건 존재한다 | 할일 목록 조회 요청을 한다 | 본인의 할일 5건만 반환된다 (타인의 할일 미포함) |
| 2 | 인증된 사용자가 기한초과 필터를 선택한다 | 할일 목록 조회 요청을 한다 | 종료일자 < 오늘 이고 상태가 완료가 아닌 할일만 반환된다 |
| 3 | 인증된 사용자가 특정 카테고리 필터를 선택한다 | 할일 목록 조회 요청을 한다 | 해당 카테고리에 속한 본인의 할일만 반환된다 |

### UC-06. 할일 수정

- **목적:** 기존 할일 내용 변경
- **흐름:** 할일 선택 → 편집 화면 진입 → 항목 수정 → 저장
- **관련 규칙:** BR-402, BR-403

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|

| 2 | 인증된 사용자가 타인 소유의 할일 ID로 수정을 시도한다 | 할일 수정 요청을 한다 | 오류 응답(권한 없음)을 반환한다 |
| 3 | 인증된 사용자가 종료일자를 시작일자보다 이전 날짜로 수정한다 | 할일 수정 요청을 한다 | 오류 응답(종료일자가 시작일자보다 이전)을 반환한다 |

### UC-08. 카테고리 생성
- **액터:** 인증된 사용자
- **목적:** 할일 분류를 위한 새 카테고리 생성
- **흐름:** 카테고리 이름 입력 → 중복 확인 → 저장
- **관련 규칙:** BR-303, BR-305

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 인증된 사용자가 본인이 보유하지 않은 새 카테고리 이름을 입력한다 | 카테고리 생성 요청을 한다 | 카테고리가 생성되고 성공 응답을 반환한다 |
| 2 | 인증된 사용자가 이미 본인이 보유한 카테고리와 동일한 이름을 입력한다 | 카테고리 생성 요청을 한다 | 오류 응답(카테고리 이름 중복)을 반환한다 |
| 3 | 인증되지 않은 상태이다 | 카테고리 생성 요청을 한다 | 오류 응답(인증 필요)을 반환한다 |

### UC-09. 카테고리 수정
- **액터:** 인증된 사용자
- **목적:** 기존 카테고리 이름 변경
- **흐름:** 카테고리 선택 → 이름 수정 → 저장
- **관련 규칙:** BR-302, BR-303, BR-305

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 인증된 사용자가 본인 소유의 카테고리를 선택하고 중복되지 않는 이름으로 변경한다 | 카테고리 수정 요청을 한다 | 카테고리 이름이 변경되고 성공 응답을 반환한다 |
| 2 | 인증된 사용자가 '기본' 카테고리를 수정하려 한다 | 카테고리 수정 요청을 한다 | 오류 응답(기본 카테고리 수정 불가)을 반환한다 |
| 3 | 인증된 사용자가 타인 소유의 카테고리를 수정하려 한다 | 카테고리 수정 요청을 한다 | 오류 응답(권한 없음)을 반환한다 |

### UC-10. 카테고리 삭제
- **액터:** 인증된 사용자
- **목적:** 불필요한 카테고리 제거
- **흐름:** 카테고리 선택 → 삭제 요청 → 확인 → 삭제 완료 (속한 할일은 '기본' 카테고리로 이동)
- **관련 규칙:** BR-302, BR-304, BR-305

**수용 기준:**

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 인증된 사용자가 본인 소유의 카테고리(할일 3건 포함)를 선택한다 | 카테고리 삭제 요청을 한다 | 카테고리가 삭제되고, 속해 있던 할일 3건의 카테고리는 '기본'으로 변경된다 |
| 2 | 인증된 사용자가 '기본' 카테고리를 삭제하려 한다 | 카테고리 삭제 요청을 한다 | 오류 응답(기본 카테고리 삭제 불가)을 반환한다 |
| 3 | 인증된 사용자가 타인 소유의 카테고리를 삭제하려 한다 | 카테고리 삭제 요청을 한다 | 오류 응답(권한 없음)을 반환한다 |

## 7. 도메인 용어 정의 (유비쿼터스 언어)

| 용어 | 정의 |
|------|------|
| 사용자 (User) | 회원가입 후 로그인하여 앱을 사용하는 주체 |
| 인증 (Authentication) | 이메일과 비밀번호로 본인임을 확인하는 절차 |
| 할일 (Todo) | 사용자가 등록한 수행 과제 단위 |
| 카테고리 (Category) | 할일을 분류하는 그룹 단위 |
| 기본 카테고리 | 카테고리 미지정 시 자동 적용되는 시스템 기본 카테고리 |
| 시작일자 | 할일을 시작하는 날짜 (당일 포함, ≤ 기준) |
| 종료일자 | 할일을 완료해야 하는 목표 날짜 (당일 포함, ≥ 기준) |
| 미시작 | 시작일자가 오늘보다 미래(시작일자 > 오늘)이거나, 시작일자 미지정이고 완료되지 않은 할일의 상태 |
| 진행중 | 시작일자 ≤ 오늘 이고 종료일자 ≥ 오늘 이며 완료되지 않은 할일의 상태 |
| 완료 | 사용자가 명시적으로 완료 처리한 할일의 상태 |
| 기한초과 | 종료일자 < 오늘(어제 이전) 이고 완료되지 않은 할일에 동적으로 계산되는 Derived 속성 |
| Derived Attribute | 데이터베이스에 저장하지 않고 조회 시점에 동적으로 계산되는 속성 |
| 오늘 (today) | 서버 기준 현재 날짜 (시각 정보 제외, 날짜만 비교) |
| 테마 (Theme) | 사용자가 선택하는 UI 색상 모드 (light 또는 dark), 사용자 정보에 저장됨 |
| 언어 (Language) | 사용자가 선택하는 UI 표시 언어 (ko: 한국어, en: 영어), 사용자 정보에 저장됨 |
| 다국어 처리 (i18n) | 앱 내 UI 텍스트를 선택된 언어로 표시하는 국제화(Internationalization) 기능 |
