## 1. 제품 개요

### 1.1 제품 비전

인증된 사용자가 개인별 할일을 등록·조회·수정·삭제하고, 카테고리 분류 및 시작일/종료일 기반 일정 관리를 통해 개인 업무와 일상을 체계적으로 관리할 수 있는 웹 기반 할일 관리 애플리케이션이다.

### 1.2 목표 사용자

- 학생 및 20~50대 직장인
- 개인 단위로 할일과 일정을 관리해야 하는 사람

### 1.3 해결 문제

| 문제 | 해결 방향 |
|------|----------|
| 개인 인증 기반 할일 관리 부재 | 이메일/비밀번호 인증을 통해 사용자별 독립된 할일 데이터 관리 |
| 일정 기간 관리 기능 부재 | 시작일자/종료일자 입력 및 기한초과 자동 판별 기능 제공 |

## 2. 범위 (Scope)

### In Scope

- 회원가입 / 로그인 / 내 정보 수정
- 할일 등록 / 목록 조회(필터링) / 수정 / 삭제
- 카테고리 생성 / 수정 / 삭제
- 기한초과 여부 자동 계산 (Derived)
- 반응형 웹 UI (모바일 대응)
- 다국어 처리 (한국어 / 영어)
- 테마 모드 선택 (Dark Mode / Light Mode)

### Out of Scope

- 소셜 로그인 (Google, Kakao 등)
- 이메일 인증 / 비밀번호 찾기
- 팀/공유 기능 (할일 공유, 협업)
- 알림 / 푸시 기능
- 파일 첨부
- 접근성(Accessibility) 대응
- 네이티브 모바일 앱 (iOS / Android)
- 관리자 기능

## 8. 데이터 모델 요약

### users 테이블

| 컬럼 | 타입 | 제약 |
|------|------|------|
| id | SERIAL / UUID | PK |
| email | VARCHAR | UNIQUE, NOT NULL |
| password | VARCHAR | NOT NULL (해시 저장) |
| name | VARCHAR | NOT NULL |
| theme | VARCHAR | NOT NULL, DEFAULT 'light' (light/dark) |
| language | VARCHAR | NOT NULL, DEFAULT 'ko' (ko/en) |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() |

### categories 테이블

| 컬럼 | 타입 | 제약 |
|------|------|------|
| id | SERIAL / UUID | PK |
| name | VARCHAR | NOT NULL |
| user_id | FK → users.id | NULL 허용 ('기본' 카테고리는 NULL) |

> '기본' 카테고리는 user_id = NULL 인 시스템 공통 레코드로 관리한다.
> (user_id, name) 조합에 UNIQUE 제약 적용하여 사용자별 이름 중복 방지

### todos 테이블

| 컬럼 | 타입 | 제약 |
|------|------|------|
| id | SERIAL / UUID | PK |
| title | VARCHAR | NOT NULL |
| description | TEXT | NULL 허용 |
| start_date | DATE | NULL 허용 |
| end_date | DATE | NULL 허용 |
| status | VARCHAR | NOT NULL (미시작/진행중/완료) |
| category_id | FK → categories.id | NOT NULL |
| user_id | FK → users.id | NOT NULL |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() |
| updated_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() |

> 기한초과 여부(is_overdue)는 저장 컬럼 없이 조회 시 동적 계산 (end_date < CURRENT_DATE AND status ≠ '완료')

## 10. 제약사항 및 가정

| 구분 | 내용 |
|------|------|
| 가정 | 모든 날짜 비교는 서버 기준 날짜(CURRENT_DATE)를 사용한다 |
| 가정 | '기본' 카테고리는 DB 초기 데이터(seed)로 미리 생성한다 |
| 가정 | 로그아웃은 클라이언트 측 토큰 삭제로 처리하며, 서버 측 토큰 무효화는 구현하지 않는다 |
| 가정 | 페이지네이션은 1차 구현 범위에서 제외하며, 목록 조회는 전체 반환으로 처리한다 |
| 가정 | 지원 언어는 한국어(ko)와 영어(en)로 한정하며, 추가 언어 확장은 Out of Scope이다 |
| 가정 | 테마 및 언어 설정은 users 테이블에 저장되며, 로그인 후 API 응답값으로 클라이언트에 적용한다 |
| 제약 | Prisma를 비롯한 ORM은 사용하지 않으며, pg 라이브러리로 직접 SQL을 작성한다 |
| 제약 | 모바일 앱(iOS/Android) 개발은 Out of Scope이며, 반응형 웹으로 대체한다 |
| 제약 | 2일 일정에 맞게 핵심 기능 우선 구현하며, 부가 기능(알림, 첨부파일 등)은 포함하지 않는다 |
