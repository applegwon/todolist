## 공통 컴포넌트

### 헤더 (Header) — 인증된 사용자 화면 공통

```
+------------------------------------------------------------------+
| TodoList                       [카테고리] [내 정보] [로그아웃]   |
+------------------------------------------------------------------+
```

- **로고/앱명 (TodoList)**: 클릭 시 할일 목록 화면(S-03)으로 이동
- **카테고리 링크**: 카테고리 관리 화면(S-06)으로 이동
- **내 정보 링크**: 내 정보 수정 화면(S-07)으로 이동
- **로그아웃 버튼**: 클라이언트 측 JWT 토큰 삭제 후 로그인 화면(S-02)으로 이동

## S-01: 회원가입 화면

**화면 목적**: 신규 사용자가 이메일, 비밀번호, 이름을 입력하여 계정을 생성한다.

**접근 조건**: 인증 불필요 (비인증 사용자 접근 가능)

- **회원가입 버튼 클릭**: POST /api/auth/signup 호출

- **로그인 링크 클릭**: 로그인 화면(S-02)으로 이동

### 관련 UC

- UC-101: 이메일, 비밀번호, 이름을 입력하여 회원가입
- UC-102: 이메일 중복 시 오류 메시지 표시

- UC-205: 로그인 시 저장된 테마/언어 자동 적용

## S-03: 할일 목록 화면

**화면 목적**: 인증된 사용자의 할일 목록을 표시하고, 카테고리/상태/기한초과 필터로 조건별 조회를 제공한다.

**접근 조건**: 인증 필요

### 데스크톱 와이어프레임

```
+------------------------------------------------------------------+
| TodoList                       [카테고리] [내 정보] [로그아웃]   |
+------------------------------------------------------------------+
|                                                                  |
|  [ 필터 영역 ]                                   [+ 할일 추가]  |
|  +------------------------------------------------------------+  |
|  | 카테고리: [전체 v]  상태: [전체 v]  [기한초과만 보기 □]    |  |
|  +------------------------------------------------------------+  |
|                                                                  |
|  +------------------------------------------------------------+  |
|  | [기본]  제목: 중간고사 공부          상태: 미시작           |  |
|  |         시작일: —  종료일: —                                |  |
```

### UI 구성 요소 목록

| 요소 | 역할 |
|------|------|
| 헤더 | 공통 헤더 컴포넌트 |
| 카테고리 필터 드롭다운 | 전체/카테고리별 필터, GET /api/todos?category=id 호출 |
| 상태 필터 드롭다운 | 전체/미시작/진행중/완료 필터, GET /api/todos?status=값 호출 |
| 기한초과만 보기 체크박스 | 기한초과 할일만 표시, GET /api/todos?overdue=true 호출 |
| 할일 추가 버튼 | 할일 등록 화면(S-04)으로 이동 |
| 할일 카드 목록 | 할일 제목, 카테고리 배지, 상태, 시작일/종료일, 기한초과 배지 표시 |
| 수정 버튼 | 해당 할일의 수정 화면(S-05)으로 이동 |
| 삭제 버튼 | 확인 후 DELETE /api/todos/:id 호출, 목록 즉시 갱신 |
| 빈 목록 안내 문구 | 필터 조건에 맞는 할일 없을 때 표시 |

### 사용자 인터랙션

- **카테고리 필터 변경**: GET /api/todos?category={id} 재호출, 목록 갱신
- **상태 필터 변경**: GET /api/todos?status={값} 재호출, 목록 갱신
- **기한초과만 보기 체크**: GET /api/todos?overdue=true 재호출, 목록 갱신
- **할일 추가 버튼 클릭**: 할일 등록 화면(S-04)으로 이동
- **수정 버튼 클릭**: 할일 수정 화면(S-05)으로 이동
- **삭제 버튼 클릭**: 확인 다이얼로그 표시 → 확인 시 DELETE /api/todos/:id 호출 → 목록 갱신

### 관련 UC

- UC-404: 본인 할일 목록만 조회
- UC-405: 카테고리/상태/기한초과 필터링
- UC-406: 기한초과 여부 동적 표시 (is_overdue 배지)
- UC-409: 할일 삭제

  - 성공(200): 모달 닫기, 목록 즉시 갱신
  - 실패(409, 이름 중복): 오류 메시지 표시
- **삭제 버튼 클릭**: 확인 다이얼로그 표시 → 확인 시 DELETE /api/categories/:id 호출 → 목록 갱신
